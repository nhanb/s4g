About

This is a sample website to demonstrate some features of s4g. It also doubles as a lazy end-to-end test suite until the core design stablizes. Its source code, as well as output, is in the /docs/ dir.

The articles themselves are just, uh, colorful lorem ipsums. Don’t worry too much about their actual content.

Sales pitch

s4g aims to assist, not obscure. All it does is reading .dj files and spitting out .htmls, leaving unrelated files alone. This means the user is free to structure their web directory however they want, and relative links work out of the box.

Features

So far we have:

  • Autogenerated home index
  • RSS Atom feed
  • Per-article custom theme

Less obvious features:

  • Flexible root dir (the finished site can be served from a subfolder)
  • Livereload
  • In-browser error messages
